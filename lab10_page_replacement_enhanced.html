Section 1 — Foundation

Why Do We Need Page Replacement?

🏠 The Hotel Analogy

Imagine a hotel (RAM) with only 3 rooms (frames). Guests (pages) keep checking in. When all rooms are full and a new guest arrives, the manager must evict someone to make room. Which guest do you evict? That's the page replacement problem. The goal: evict the guest who will be back last — minimising disruption (page faults).

The Core Problem

In a system using demand paging, pages are loaded into physical memory (frames) only when needed. Physical memory is finite. When a referenced page is not in memory (page fault), the OS must:

  1. Find a free frame — or pick a victim frame to evict
  2. If victim page is dirty (modified), write it to disk
  3. Load the needed page into the freed frame
  4. Restart the faulted instruction

Page Fault

CPU accesses a virtual page that isn't in physical memory. Triggers OS interrupt. Expensive: ~10ms vs ~100ns for a memory hit. The fewer faults, the better.

Page Hit

Page is already in a frame. Access proceeds at full memory speed. Hit ratio = hits / total accesses. We want this as high as possible.

Why Not Just Add More RAM?

Even with large RAM, workloads can exceed physical memory. Virtual memory lets programs address far more memory than physically exists — page replacement is the mechanism that makes this work. Without it, programs would simply crash when memory ran out.

Section 2 — Foundation

Key Concepts

Frame

Fixed-size block in physical memory. Number of frames = physical memory / page size. Pages are loaded into frames.

Page

Fixed-size block of logical address space. Same size as frame. Identified by page number in the reference string.

Reference String

Sequence of page numbers accessed by a process over time. e.g. 7,0,1,2,0,3,0,4...

Evaluating Algorithms

We evaluate page replacement algorithms by counting page faults on a given reference string with a fixed number of frames. Fewer faults = better algorithm.

Hit Ratio = (Total References − Page Faults) / Total References

Types of Page Replacement

Global Replacement

A process can steal a frame from any other process. Better overall throughput, but a process can't control its own fault rate. Used by most OSes.

Local Replacement

A process can only replace one of its own frames. Consistent per-process performance, but can cause underutilisation if some processes have spare frames.

Section 3 — Algorithms

FIFO — First In, First Out

Simplest
First In, First Out

Evict the page that has been in memory the longest. Think of it as a queue — the oldest resident leaves first. Easy to implement; not always efficient.

How it Works

Maintain a queue of pages in order of arrival. When a frame is needed, remove the page at the front (oldest). Insert the new page at the back.

  • Initialise: All frames empty. Queue = [].
  • On each page reference: Is the page already in a frame? → HIT, no action.
  • Page not in frames (FAULT): If empty frame available, load page there. If all frames full, evict the front of the FIFO queue.
  • Add to queue: Push new page to back of queue. Increment fault counter.

⚠️ Bélády's Anomaly Applies to FIFO

Counterintuitively, FIFO can produce more page faults with more frames. This defect is known as Bélády's Anomaly. Only FIFO (and some other algorithms) suffer from it. OPT and LRU do not.

Pseudocode 
// FIFO Page Replacement
function fifo(pages[], n_frames):
    frames = empty queue (max size n_frames)
    faults = 0

    for each page in pages:
        if page in frames:
            // HIT — no action
        else:
            faults++
            if frames.size < n_frames:
                frames.enqueue(page)    // free frame available
            else:
                evicted = frames.dequeue()  // remove oldest
                frames.enqueue(page)        // add newest

    return faults
Section 4 — Algorithms

Optimal Page Replacement (OPT)

Theoretical Best
Optimal / Bélády's Algorithm

Evict the page that will not be used for the longest time in the future. Provably optimal — fewest possible faults. Cannot be implemented in real systems (requires future knowledge) but serves as a benchmark.

How it Works

On a page fault, look ahead in the reference string. For each page currently in a frame, find when it will next be used. Evict the page with the furthest next use (or never used again — evict that first).

  • Page fault occurs. Frames are all full.
  • For each page in frames: Scan forward in the reference string. Find the index of its next occurrence.
  • Evict the page with the largest "next use" index. If a page never appears again, it's the best candidate to evict.
  • Load new page into the freed frame.

Why Study OPT if it's Unimplementable?

OPT gives us a lower bound on page faults. If your FIFO gives 15 faults and OPT gives 9, you know the room for improvement. It also helps evaluate how close to optimal other algorithms are. LRU is close to OPT in practice.

Pseudocode 
function optimal(pages[], n_frames):
    frames = []
    faults = 0

    for i = 0 to len(pages)-1:
        if pages[i] in frames:
            continue  // HIT

        faults++
        if len(frames) < n_frames:
            frames.add(pages[i])
        else:
            // Find which frame's page is used furthest in future
            victim = -1; maxDist = -1
            for each f in frames:
                nextUse = find_next_use(pages, i+1, f)  // index of next use
                if nextUse == NEVER:
                    victim = f; break      // best candidate
                if nextUse > maxDist:
                    maxDist = nextUse; victim = f
            frames.replace(victim, pages[i])

    return faults
Section 5 — Algorithms

LRU — Least Recently Used

Most Practical
Least Recently Used

Evict the page that was used least recently (longest time since last access). Uses the past to predict the future — a good approximation of OPT. Does not suffer from Bélády's anomaly.

Key Insight

LRU exploits temporal locality: if a page was used recently, it's likely to be used again soon. Pages unused for a long time are less likely to be needed. This is usually true for real programs (loops, working sets).

  • On each reference: Update a timestamp or move page to "most recently used" position.
  • On page fault: Evict the page with the smallest timestamp (oldest last-use time).
  • Implementation: Use a doubly-linked list + hash map for O(1) access. Or use a timestamp counter.

LRU Implementation Cost

True LRU requires updating page order on every memory reference — potentially millions per second. Hardware support (reference bits, page table bits) is needed. In practice, OSes use approximations of LRU rather than exact LRU.

Section 6 — Algorithms

LRU Approximation Algorithms

Exact LRU is expensive. These algorithms approximate it using hardware reference bits that are cheaper to maintain.

1. Reference Byte Algorithm (8-bit Counter)

How it works

  • Each page has an 8-bit shift register, initialised to 00000000
  • On a periodic timer interrupt (e.g. every 100ms): shift all counters right by 1 bit
  • Set the high-order bit to 1 if the page was referenced since last timer, 0 otherwise
  • Evict the page with the smallest counter value (numerically least = least recently used)
Page A: 11000100 → recently used twice
Page B: 01110111 → used frequently but not recently
Page C: 00000000 → not used in 8 intervals → EVICT

2. Second-Chance (Clock) Algorithm

How it works

A FIFO queue but each page has a reference bit (R). On replacement:

  • Inspect the front page of the queue
  • If R = 0 → evict it (old and not recently used)
  • If R = 1 → clear R to 0, move page to back of queue ("give it a second chance"), advance pointer
  • A page with R=1 will be evicted on the second pass if still not used again
  • Circular queue traversal looks like a clock hand sweeping — hence "clock algorithm"

3. Enhanced Second-Chance (NRU) — 2 Bits

Reference bit (R) + Modified bit (M)

Four classes of pages, in order of preference for eviction:

Class(R, M)MeaningEviction Priority
0(0, 0)Not recently used, not modifiedBest — evict first
1(0, 1)Not recently used, modified (dirty)Must write to disk first
2(1, 0)Recently used, cleanProbably needed again
3(1, 1)Recently used, modifiedWorst — evict last
Section 7 — Algorithms

Counting-Based Algorithms

Keep a counter of how many times each page has been referenced. Two opposite strategies:

LFU — Least Frequently Used

Evict the page with the lowest reference count.

Problem: A page heavily used during program startup accumulates a high count and won't be evicted — even after startup when it's no longer needed. Counts never reset.

MFU — Most Frequently Used

Evict the page with the highest reference count.

Reasoning: A newly loaded page has a small count — it probably hasn't had time to be used, so don't evict it. Unusual heuristic. Not used in mainstream OSes.

Counting Algorithms in Practice

Neither LFU nor MFU perform well in general. They're not used in modern mainstream OSes. However, LFU-like ideas appear in database buffer management and cache design where access patterns are more predictable.

The Aging Algorithm — LFU's Practical Cousin

Pure LFU never forgets old counts. The Aging algorithm fixes this by combining the reference bit approach with bit-shifting — old references naturally "age out" over time as the counter is right-shifted each interval.

How Aging Works (Per Timer Interrupt)

① Shift each page's counter right by 1 bit
② OR the reference bit into the leftmost (high-order) bit
③ Clear the reference bit
④ On replacement: evict page with lowest counter value
Example over 4 intervals (R=1 means referenced that interval):
Page A: R=1,0,0,1 → after aging: 10000001 → moderately used
Page B: R=1,1,1,0 → after aging: 01110000 → heavily used recently
Page C: R=0,0,0,0 → after aging: 00000000 → not used → EVICT

 * 2. MFU  — Most Frequently Used
 * 3. Aging — LFU variant with time-decaying counters

#include <stdio.h>
#include <string.h>
#include <limits.h>
#include <stdbool.h>

#define MAX_F  10
#define MAX_P  100
#define AGING_BITS 8   /* 8-bit shift register for aging */

/* ─────────── shared state ─────────── */
int nf, np;
int pages[MAX_P];

void print_header(const char* name) {
    printf("\n╔══════════════════════════════════╗\n");
    printf("  %-34s\n", name);
    printf("╚══════════════════════════════════╝\n");
    printf("%-5s  %-8s  %s\n", "Page", "Result", "Frames (page:count)");
    printf("------  --------  %.*s\n", 40, "----------------------------------------");
}

/* ─────────────────────────────────────
   Helper: find page in frame array
   Returns index or -1
───────────────────────────────────── */
int find_page(int fr[], int pg) {
    for (int i = 0; i < nf; i++)
        if (fr[i] == pg) return i;
    return -1;
}

/* ─────────────────────────────────────
   Print frame state: [p:count p:count …]
───────────────────────────────────── */
void show_frames(int fr[], int cnt[]) {
    printf("  [");
    for (int i = 0; i < nf; i++) {
        if (fr[i] == -1) printf(" ---");
        else             printf(" p%d:%d", fr[i], cnt[i]);
    }
    printf(" ]\n");
}

/* ═══════════════════════════════════════════════════
   LFU — evict the frame with the LOWEST frequency.
   Tie-break: evict the one loaded longest ago (FIFO).
═══════════════════════════════════════════════════ */
int lfu() {
    int fr[MAX_F], freq[MAX_F], load_time[MAX_F];
    int fc = 0, faults = 0, timer = 0;

    for (int i = 0; i < nf; i++) { fr[i]=-1; freq[i]=0; load_time[i]=0; }

    print_header("LFU (Least Frequently Used)");

    for (int i = 0; i < np; i++) {
        int pg = pages[i];
        timer++;

        int idx = find_page(fr, pg);
        if (idx != -1) {
            /* ── HIT: bump frequency ── */
            freq[idx]++;
            printf("%-5d  HIT    ", pg);
        } else {
            /* ── FAULT ── */
            faults++;
            int slot;

            if (fc < nf) {
                slot = fc++;   /* empty frame available */
            } else {
                /* Find minimum-frequency slot; FIFO tie-break */
                slot = 0;
                for (int j = 1; j < nf; j++) {
                    if (freq[j] < freq[slot] ||
                       (freq[j] == freq[slot] &&
                        load_time[j] < load_time[slot]))
                        slot = j;
                }
                printf("       └─ evict p%d (freq=%d)\n",
                       fr[slot], freq[slot]);
            }

            fr[slot]        = pg;
            freq[slot]      = 1;       /* fresh page starts at 1 */
            load_time[slot] = timer;

            printf("%-5d  FAULT  ", pg);
        }
        show_frames(fr, freq);
    }

    printf("\nPage faults : %d\n", faults);
    printf("Hit ratio   : %.1f%%\n", (float)(np-faults)/np*100);
    return faults;
}

/* ═══════════════════════════════════════════════════
   MFU — evict the frame with the HIGHEST frequency.
   Logic: newly loaded pages have low count and are
   considered "not yet used enough" — keep them.
   Tie-break: evict the one loaded longest ago.
═══════════════════════════════════════════════════ */
int mfu() {
    int fr[MAX_F], freq[MAX_F], load_time[MAX_F];
    int fc = 0, faults = 0, timer = 0;

    for (int i = 0; i < nf; i++) { fr[i]=-1; freq[i]=0; load_time[i]=0; }

    print_header("MFU (Most Frequently Used)");

    for (int i = 0; i < np; i++) {
        int pg = pages[i];
        timer++;

        int idx = find_page(fr, pg);
        if (idx != -1) {
            freq[idx]++;
            printf("%-5d  HIT    ", pg);
        } else {
            faults++;
            int slot;

            if (fc < nf) {
                slot = fc++;
            } else {
                /* Find maximum-frequency slot; FIFO tie-break */
                slot = 0;
                for (int j = 1; j < nf; j++) {
                    if (freq[j] > freq[slot] ||
                       (freq[j] == freq[slot] &&
                        load_time[j] < load_time[slot]))
                        slot = j;
                }
                printf("       └─ evict p%d (freq=%d)\n",
                       fr[slot], freq[slot]);
            }

            fr[slot]        = pg;
            freq[slot]      = 1;
            load_time[slot] = timer;
            printf("%-5d  FAULT  ", pg);
        }
        show_frames(fr, freq);
    }

    printf("\nPage faults : %d\n", faults);
    printf("Hit ratio   : %.1f%%\n", (float)(np-faults)/np*100);
    return faults;
}

/* ═══════════════════════════════════════════════════
   AGING ALGORITHM
   —————————————————————————————————————————————————
   Each page has an 8-bit counter (shift register).
   On every reference: set counter's MSB to 1.
   On every timer tick : shift counter RIGHT by 1.
   On fault: evict page with the SMALLEST counter.

   This approximates LRU but counters decay over time,
   solving LFU's "startup pollution" problem.

   Simulated here without a real timer: we treat each
   page access as one "tick" for simplicity.
═══════════════════════════════════════════════════ */
int aging() {
    int          fr[MAX_F];
    unsigned int ctr[MAX_F];    /* 8-bit shift register per frame */
    int          fc = 0, faults = 0;

    for (int i = 0; i < nf; i++) { fr[i] = -1; ctr[i] = 0; }

    print_header("Aging Algorithm (LFU + time-decay)");
    printf("  (Counter shown as 8-bit binary)\n\n");

    for (int i = 0; i < np; i++) {
        int pg = pages[i];

        /* Step 1: shift ALL counters right by 1 (age them) */
        for (int j = 0; j < nf; j++)
            ctr[j] = (ctr[j] >> 1) & 0xFF;   /* keep 8 bits */

        int idx = find_page(fr, pg);

        if (idx != -1) {
            /* Step 2a: HIT — set MSB of this page's counter */
            ctr[idx] |= 0x80;               /* set bit 7 */
            printf("%-5d  HIT    ", pg);
        } else {
            /* Step 2b: FAULT */
            faults++;
            int slot;

            if (fc < nf) {
                slot = fc++;
            } else {
                /* Evict page with MINIMUM counter value */
                slot = 0;
                for (int j = 1; j < nf; j++)
                    if (ctr[j] < ctr[slot]) slot = j;

                printf("       └─ evict p%d (ctr=%02X)\n",
                       fr[slot], ctr[slot]);
            }

            fr[slot]  = pg;
            ctr[slot] = 0x80;   /* newly loaded → MSB=1, rest 0 */
            printf("%-5d  FAULT  ", pg);
        }

        /* Print frames with binary counters */
        printf("  [");
        for (int j = 0; j < nf; j++) {
            if (fr[j] == -1) { printf(" --------"); continue; }
            printf(" p%d:", fr[j]);
            for (int b = 7; b >= 0; b--)
                printf("%d", (ctr[j] >> b) & 1);
        }
        printf(" ]\n");
    }

    printf("\nPage faults : %d\n", faults);
    printf("Hit ratio   : %.1f%%\n", (float)(np-faults)/np*100);
    return faults;
}

    printf("Number of refs   : "); scanf("%d", &np);
    printf("Page references  : ");

    int fm = mfu();

LFU vs Aging — The Key Difference

LFU counts never decay — a page accessed 100 times at startup will hoard its count forever. Aging uses a shift register so old accesses contribute progressively less to the counter value. After 8 timer intervals without access, a page's contribution from those old accesses has completely shifted out — making it a viable eviction candidate again. This is why Aging is considered a superior approximation to pure LFU.

The surprising phenomenon where increasing the number of frames causes MORE page faults — only with FIFO (and a few other policies).

With 3 frames: 9 faults. With 4 frames: 10 faults. More frames, more faults! This is Bélády's Anomaly.

Why Does This Happen?

FIFO doesn't account for how recently a page was used — it blindly evicts the oldest page. With more frames, the replacement order changes in a way that happens to evict pages that are about to be needed. FIFO has no concept of page "value."

Algorithms that SUFFER from Bélády's

  • FIFO
  • Second Chance (sometimes)

Algorithms IMMUNE to Bélády's

  • OPT (provably optimal)
  • LRU (stack algorithm)
  • All "stack algorithms"

Stack Algorithms — Why They're Anomaly-Free

An algorithm is a stack algorithm if: the set of pages in memory with n frames is always a subset of the pages in memory with n+1 frames (for any reference string). OPT and LRU have this property — adding a frame can only help. FIFO does not.

C Code Implementation

C 
/*
 * FIFO Page Replacement Algorithm
 * Simulates a FIFO queue of page frames.
 * Input: reference string, number of frames
 * Output: frame state at each step + total page faults
 */
#include <stdio.h>
#include <stdbool.h>
#include <string.h>

#define MAX_FRAMES 10
#define MAX_PAGES  50

int isInFrames(int frames[], int n, int page) {
    for (int i = 0; i < n; i++)
        if (frames[i] == page) return 1;
    return 0;
}

int fifo(int pages[], int n, int nFrames) {
    int frames[MAX_FRAMES];
    int faults = 0;
    int nextReplace = 0;  /* index of next frame to replace (FIFO pointer) */
    int frameCount = 0;    /* how many frames are currently occupied */

    memset(frames, -1, sizeof(frames));  /* -1 = empty frame */

    printf("FIFO — %d frames\n", nFrames);
    printf("%-5s %-8s %-20s %s\n", "Ref", "Frames", "", "Fault?");

    for (int i = 0; i < n; i++) {
        int page = pages[i];

        if (isInFrames(frames, nFrames, page)) {
            /* HIT — page is already in a frame */
            printf("%-5d ", page);
            for (int j = 0; j < nFrames; j++)
                frames[j] == -1 ? printf("  -") : printf("%3d", frames[j]);
            printf("   HIT\n");
        } else {
            /* FAULT — page not in memory */
            faults++;
            frames[nextReplace] = page;         /* overwrite FIFO victim */
            nextReplace = (nextReplace + 1) % nFrames;  /* advance pointer */
            if (frameCount < nFrames) frameCount++;

            printf("%-5d ", page);
            for (int j = 0; j < nFrames; j++)
                frames[j] == -1 ? printf("  -") : printf("%3d", frames[j]);
            printf("   FAULT\n");
        }
    }

    printf("\nTotal page faults = %d\n", faults);

 * Optimal Page Replacement Algorithm
 * On each fault, evict the page whose next use is furthest away.
 * Requires knowing the entire future reference string.
 */
#include <stdio.h>
#include <string.h>
#define MAX_F 10
#define INF   99999

    for(int i=0;i<nf;i++) if(f[i]==p) return i;
    return -1;

/* Find next use of page p after position cur in pages[] */
int nextUse(int pages[], int n, int cur, int p) {
    for(int i = cur; i < n; i++)
        if(pages[i] == p) return i;
    return INF;  /* never used again — best to evict */
}

int optimal(int pages[], int n, int nf) {
    int frames[MAX_F], faults = 0, fc = 0;
    memset(frames, -1, sizeof(frames));

    printf("OPT — %d frames\n", nf);
    printf("%-5s %-24s %s\n", "Ref", "Frames", "Fault?");

    for(int i = 0; i < n; i++) {
        int pg = pages[i];

        if(isIn(frames, nf, pg) != -1) {
            printf("%-5d", pg);
            for(int j=0;j<nf;j++) printf("%3d",frames[j]); printf("  HIT\n");
            continue;
        }

        faults++;
        if(fc < nf) {
            /* Still have empty frames — just fill them */
            frames[fc++] = pg;
        } else {
            /* All frames full — find optimal victim */
            int victim = 0, maxDist = -1;
            for(int j = 0; j < nf; j++) {

        printf("%-5d", pg);
        for(int j=0;j<nf;j++)
            frames[j]==-1 ? printf("  -") : printf("%3d",frames[j]);
        printf("  FAULT\n");
    }
    printf("Total faults = %d  Hit ratio = %.2f%%\n",
           faults, (float)(n-faults)/n*100);
    return faults;
}

/*
 * LRU Page Replacement Algorithm
 * Uses a "last used time" counter for each frame.
 * Evicts the frame with the smallest (oldest) last-use time.
 */
#include <stdio.h>
#include <string.h>
#define MAX_F 10

int lru(int pages[], int n, int nf) {
    int frames[MAX_F];      /* current pages in frames */
    int lastUsed[MAX_F];    /* timestamp of last use for each frame slot */
    int faults = 0, fc = 0, timer = 0;

    memset(frames,   -1, sizeof(frames));
    memset(lastUsed, -1, sizeof(lastUsed));

    printf("LRU — %d frames\n", nf);
    printf("%-5s %-24s %s\n", "Ref", "Frames", "Fault?");

    for(int i = 0; i < n; i++) {
        int pg = pages[i]; timer++;
        int hitIdx = -1;

        /* Check if page is already in a frame */
        for(int j = 0; j < nf; j++)
            if(frames[j] == pg) { hitIdx = j; break; }

        if(hitIdx != -1) {
            /* HIT — update last used time */
            lastUsed[hitIdx] = timer;
            printf("%-5d", pg);
            for(int j=0;j<nf;j++) printf("%3d",frames[j]);
            printf("  HIT\n");
        } else {
            faults++;
            int slot;

            if(fc < nf) {
                /* Free frame available */
                slot = fc++;
            } else {
                /* Evict the page with smallest lastUsed (LRU) */
                int minTime = lastUsed[0], minIdx = 0;
                for(int j = 1; j < nf; j++) {
                    if(lastUsed[j] < minTime) {
                        minTime = lastUsed[j];
                        minIdx  = j;
                    }
                }
                slot = minIdx;
            }
            frames[slot]   = pg;
            lastUsed[slot] = timer;

            printf("%-5d", pg);
            for(int j=0;j<nf;j++)
                frames[j]==-1 ? printf("  -") : printf("%3d",frames[j]);
            printf("  FAULT\n");
        }
    }
    printf("Total faults = %d  Hit ratio = %.2f%%\n",
           faults, (float)(n-faults)/n*100);
    return faults;
}

int main() {
    int pages[] = {7,0,1,2,0,3,0,4,2,3,0,3,2,1,2,0,1,7,0,1};
    int n = sizeof(pages)/sizeof(pages[0]);
    lru(pages, n, 3);
    return 0;
}

Common Mistakes

✗
Thinking more frames always = fewer faults
True for LRU and OPT (stack algorithms). FALSE for FIFO — Bélády's anomaly means more frames can cause more faults. A very common trick question.
✗
Counting a hit as a fault in FIFO
If the page is already in a frame — even if it's the "oldest" — it is a HIT, not a fault. FIFO only evicts on a fault when all frames are full.
✗
LRU evicts the page referenced longest ago vs FIFO evicts the oldest in memory
These are different! LRU tracks when a page was last used (any reference). FIFO tracks when it was first loaded. A page loaded long ago but used recently stays in LRU but gets evicted by FIFO.
✗
OPT evicting the wrong page
Always look forward from the current position + 1, not the current position. If a page never appears again in the future, evict it first — don't scan to end looking for the largest index.
✓
The first few references are always faults (cold start)
Every algorithm starts with empty frames. The first n_frames unique pages are always faults regardless of algorithm. Don't waste time second-guessing these — they're compulsory faults (also called "cold misses").
✗
Mixing up LFU and LRU
LRU = Least Recently Used (time-based). LFU = Least Frequently Used (count-based). They produce very different results. LRU is used in real OSes; LFU mostly in databases.
Section 14 — Exam Prep

Viva / Exam Questions

Short Answer

What is a page fault? ▾
A page fault occurs when a process references a virtual page that is not currently loaded in physical memory (not in any frame). The OS handles it by loading the page from disk into a free or replaced frame, then restarting the instruction.
Why is OPT not used in real systems? ▾
OPT requires knowing the future reference string — which pages will be accessed next and when. An OS cannot predict future accesses at runtime. OPT is used as a theoretical benchmark to measure how close real algorithms are to the ideal.
What is the difference between FIFO and Second-Chance? ▾
Both use a queue and evict in FIFO order. The difference: Second-Chance checks the reference bit of the oldest page. If R=1 (recently used), it clears R to 0 and gives the page another rotation (a "second chance"). If R=0, it evicts immediately. Second-Chance avoids evicting recently-used pages, making it more like LRU.
What is temporal locality and why does LRU exploit it? ▾
Temporal locality is the observation that if a memory location was accessed recently, it is likely to be accessed again soon (loops, variables, function calls exhibit this). LRU exploits this by keeping recently-used pages in memory and evicting the ones unused for the longest time — predicting that old pages are less likely to be needed.
Define hit ratio and page fault rate. ▾
Hit ratio = (Total references − Page faults) / Total references
Page fault rate = Page faults / Total references
They sum to 1. A higher hit ratio means fewer expensive disk accesses and better performance.

Trick / Long Questions

🎯 TRICK: With FIFO and 4 frames, can we get MORE faults than with 3 frames? Show an example. ▾
Yes — this is Bélády's Anomaly. Example: reference string 1,2,3,4,1,2,5,1,2,3,4,5.
With 3 frames: 9 page faults.
With 4 frames: 10 page faults (more!).
This happens because adding a frame changes the FIFO eviction order in a way that happens to evict pages that are about to be needed. LRU and OPT are immune because they are stack algorithms.
🎯 What is a stack algorithm? Give two examples. ▾
A replacement algorithm is a stack algorithm if: for any reference string and any n, the set of pages in memory with n frames is always a subset of the set with n+1 frames. Formally: S(n) ⊆ S(n+1) for all reference strings.
Examples: OPT and LRU. Stack algorithms are provably immune to Bélády's anomaly — adding a frame can never increase page faults. FIFO is NOT a stack algorithm.
🎯 Why doesn't thrashing occur with enough frames? ▾
Thrashing occurs when a process doesn't have enough frames to hold its working set (the set of pages actively being used in the current phase). The process constantly page faults, spending more time on I/O than on actual computation. If the OS allocates frames ≥ working set size, the hot pages stay in memory, faults become rare, and thrashing stops. This is why the working set model is used in practice to guide frame allocation.
🎯 LRU gives fewer faults than FIFO — is this always true? ▾
No — not always. For specific reference strings, FIFO can match or even beat LRU. However, LRU is generally better in practice because it uses recency information. LRU also has a theoretical guarantee: it never performs worse than OPT by more than a constant factor under certain conditions. The lab manual example (20 refs, 3 frames) gives FIFO=15 vs LRU=12, which is the typical case.

New Topic — Foundation

Demand Paging & Virtual Memory Deep Dive

📦

Demand paging is the mechanism that makes virtual memory real. Instead of loading an entire process into RAM at startup, the OS loads pages only when they're first accessed. This is the foundation on which every page replacement algorithm sits.

Without demand paging, page replacement would never be needed — because you'd only run programs that fit entirely in RAM.

How a Page Fault is Handled — Step by Step

1. CPU generates a virtual address
The MMU checks the page table. The valid bit for this page is 0 (not in memory).
2. Page fault trap fires
Hardware raises an interrupt. The OS page fault handler takes control. The faulting instruction is saved so it can be restarted.
3. OS validates the reference
Is this a valid address (within process address space)? If not → segmentation fault, process killed. If yes → proceed.
4. Find a free frame
OS checks the free frame list. If one exists, use it. If not, the page replacement algorithm picks a victim frame to evict.
5. Evict victim (if dirty)
If the victim page was modified (dirty bit set), write it back to disk (swap space). If clean, just overwrite — no write needed.
6. Load requested page
Issue a disk I/O to bring the needed page into the freed frame. The process is blocked (sleeping) while I/O completes — another process runs.
7. Update page table
Set the valid bit to 1, update the frame number in the page table entry. Invalidate TLB entry for this page (TLB shootdown).
8. Restart faulting instruction
The process resumes. The instruction that caused the fault is re-executed. This time, the page is in memory — no fault.

💡 The Dirty Bit Optimisation

Each page table entry has a dirty/modified bit. Set by hardware whenever a page is written to. On eviction: if dirty → write to disk (expensive). If clean → just throw it away (fast — we can reload from disk if needed). This is why dirty page eviction costs two disk operations (one write + potential future read) while clean eviction costs at most one.

Pure Demand Paging vs Pre-paging

Pure Demand Paging

Never bring a page into memory until it's requested. Minimal initial overhead. But process startup triggers many faults (all first accesses are faults). Most OSes use this as the default.

Pre-paging

Bring in some pages before they're needed — e.g., load several sequential pages when one is faulted (spatial locality). Reduces fault count but may waste I/O on pages never used. Used for process startup in Linux (madvise() hint).

Copy-on-Write (COW)

When a process calls fork(), rather than duplicating all pages immediately, the OS marks both parent and child's page table entries as read-only and shared. When either tries to write, a fault fires, the OS copies just that one page, and both get independent copies. This is Copy-on-Write — fork() followed immediately by exec() never copies any pages at all.

Effective Access Time (EAT) Formula

Let p = page fault rate (0 ≤ p ≤ 1), m = memory access time, t = page fault service time (disk).

EAT = (1 − p) × m + p × t

Example: m = 200ns, t = 8ms, p = 0.001 (1 in 1000 faults)
EAT = 0.999×200 + 0.001×8,000,000 = 199.8 + 8000 ≈ 8.2µs — 40× slower than no faults!

New Topic — Critical Concept

Thrashing Always in Exams

🔥

Thrashing is when a process spends more time swapping pages in and out than actually executing useful work. CPU utilisation crashes. The system appears frozen. Understanding thrashing is one of the most important (and most tested) topics in OS memory management.

How Thrashing Develops

The Vicious Cycle

CPU utilisation drops → OS thinks CPU is underused → loads more processes into memory → each process gets fewer frames → page fault rate skyrockets for all processes → CPU spends all its time handling faults → utilisation drops further. The OS keeps making it worse by adding more processes.

Here's what the CPU utilisation curve looks like as you increase the degree of multiprogramming:

Why Does Thrashing Happen?

Every process has a working set — the set of pages it actively uses at any given time. If the total working set size of all processes exceeds the total physical frames, thrashing is inevitable.

Thrashing condition: Σ WSS(Pᵢ) > Total Physical Frames

Solutions to Thrashing

Working Set Model

Track each process's working set. Only run a process if all its working set fits in available frames. Suspend processes that can't fit — don't let them thrash. (See next section for details.)

Page Fault Frequency (PFF)

Monitor each process's page fault rate. Too high → give it more frames. Too low → take some frames away. If no free frames and fault rate still too high → suspend a process.

Swapping

Suspend entire processes and swap them out to disk. This frees up all their frames for other processes. The suspended process is brought back when more memory becomes available.

Reduce Multiprogramming

Simply run fewer processes simultaneously. Each gets more frames. Simple but limits utilisation. Modern Linux uses a combination of PFF and smart swapping (via kswapd).

Thrashing in the Real World

You've experienced thrashing when your computer with 4GB RAM had 20 browser tabs, a video editor, and a game open. The HDD light was stuck on. Everything was frozen. That was thrashing. Modern SSDs make it less catastrophic (disk I/O is faster), but the CPU still spends most time on fault handling rather than useful work.

Working Set Model

Thrashing Prevention
Working Set Model

Proposed by Peter Denning (1968). A process's working set W(t, Δ) is the set of pages referenced in the most recent Δ references (the "window"). The OS allocates at least |WSS| frames to a process. If total memory can't fit all working sets, suspend a process.

Working Set Intuition

Programs don't access memory randomly — they have locality of reference. During any phase of execution, a process actively uses only a small subset of its pages (its working set). If that subset fits in memory, the fault rate is low. If frames < working set size, every reference may fault.

Locality of Reference — Two Types

Temporal A recently used page is likely to be used again soon (loops, frequently called functions).
Spatial Pages near a recently used page are likely to be used soon (arrays, sequential code).

Working Set Trace (Δ = 4)

Reference string: 1, 2, 3, 4, 2, 1, 5, 2, 1, 2, 3, 4, 5

t=4 (Δ=4)→W={1,2,3,4}last 4 refs: 1,2,3,4
t=6 (Δ=4)→W={1,2,3,4}refs: 3,4,2,1
t=7 (Δ=4)→W={1,2,4,5}refs: 4,2,1,5 → page 3 left WS
t=9 (Δ=4)→W={1,2,5}refs: 1,5,2,1 → pages 3,4 left WS
t=11 (Δ=4)→W={1,2,3}refs: 2,1,2,3 → pages 4,5 entering

Choosing Δ (the Window Size)

  • Δ too small: Working set misses pages that are actually in active use. Too many unnecessary faults.
  • Δ too large: Working set includes stale pages from old phases. Wastes frames, reduces multiprogramming.
  • Δ = ∞: Working set = entire program. Same as loading everything upfront.
  • In practice: Δ is typically chosen to span a few "phases" of the program (hundreds to thousands of references).

WSM Policy

D = Σ|WSSᵢ| = total demand for frames.
If D < total frames → all processes run happily.
If D > total frames → select a process to suspend, free its frames, reduce D.

Approximating WSM

Exact working set tracking is expensive. OSes approximate using reference bits + periodic timer interrupts. The reference byte algorithm (8-bit shift register per page) is a common approximation.

New Topic — Advanced

Frame Allocation Strategies

Page replacement tells us which page to evict. Frame allocation tells us how many frames to give each process. Both matter — giving too few frames to a process guarantees thrashing.

Minimum Frame Requirement

Every process needs a minimum number of frames determined by the instruction set architecture. Why? Some instructions reference multiple memory locations. For example, an indirect addressing mode instruction may touch 3–4 pages in a single instruction. If the process has fewer frames than the instruction needs, it will fault on every instruction — a livelock.

Min Frames Example

IBM 370 MOVE instruction: can reference 6 different pages simultaneously (opcode, two operand addresses, two source pages, one destination page). This process needs at least 6 frames or it can never make progress.

Equal vs Proportional Allocation

Equal Allocation

If there are m frames and n processes:

Frames per process = ⌊m / n⌋
  • Simple, fair in the raw sense
  • Ignores that process sizes differ wildly
  • A tiny 10-page process gets the same as a 200-page process
  • Not used in modern OSes

Proportional Allocation

Allocate frames proportional to process size sᵢ:

framesᵢ = (sᵢ / Σsⱼ) × m
  • Larger processes get more frames
  • Fairer and more efficient
  • Can weight by priority instead of size
  • More commonly used

Priority Allocation

Instead of weighting by size, weight by process priority. A high-priority process (e.g., a real-time task) gets more frames. A background process gets fewer. This ensures important work doesn't thrash while low-priority work might.

/*
 * Clock (Second-Chance) Page Replacement Algorithm
 * ---------------------------------------------------
 * Uses a circular buffer of frames, each with a reference bit.
 * Clock hand sweeps; R=1 → clear and skip; R=0 → evict.
 * O(1) amortised per page access (best case). This is the
 * algorithm closest to what Linux uses internally.
 *
 * Compile: gcc clock_algorithm.c -o clock_algo
 */

#include <stdio.h>
#include <string.h>
#include <stdbool.h>

#define MAX_FRAMES 10
#define MAX_PAGES  100

/* Each frame slot has a page number + reference bit */
typedef struct {
    int  page;    /* page number, -1 = empty            */
    int  ref;     /* reference bit: 0 or 1              */
} Frame;

Frame frames[MAX_FRAMES];
int  hand = 0;   /* clock hand position                */
int  nf;         /* number of frames (set in main)     */

/* ── Check if page is in frames; set R=1 on hit ── */
bool page_hit(int page) {
    for (int i = 0; i < nf; i++) {
        if (frames[i].page == page) {
            frames[i].ref = 1;   /* mark as recently used */
            return true;
        }
    }
    return false;
}

/* ── Clock replacement: sweep until R=0 found ── */
void clock_replace(int page) {
    while (true) {
        if (frames[hand].ref == 0) {
            /* Found victim: R=0 → evict */
            if (frames[hand].page != -1)
                printf("        [evict page %d from slot %d]\n",
                       frames[hand].page, hand);
            frames[hand].page = page;
            frames[hand].ref  = 1;       /* newly loaded → R=1 */
            hand = (hand + 1) % nf;      /* advance past victim */
            return;
        } else {
            /* R=1: give second chance, clear bit, advance */
            printf("        [second chance: page %d R→0]\n",
                   frames[hand].page);
            frames[hand].ref = 0;
            hand = (hand + 1) % nf;
        }
    }
}

/* ── Print current frame state ── */
void print_frames() {
    printf("       Frames: [");
    for (int i = 0; i < nf; i++) {
        if (frames[i].page == -1)
            printf(" -");
        else
            printf("%2d(R=%d)", frames[i].page, frames[i].ref);
        if (i == hand - 1 || (hand == 0 && i == nf - 1))
            printf(" ←hand");
    }
    printf(" ]\n");
}

int main() {
    int pages[MAX_PAGES], n;

    printf("=== Clock (Second-Chance) Page Replacement ===\n\n");
    printf("Number of frames: "); scanf("%d", &nf);
    printf("Number of pages : "); scanf("%d", &n);
    printf("Page references  : ");
    for (int i = 0; i < n; i++) scanf("%d", &pages[i]);

    /* Initialise all frames as empty */
    for (int i = 0; i < nf; i++) { frames[i].page = -1; frames[i].ref = 0; }

    int faults = 0, empty_slots = nf;
    printf("\n%-6s %-8s %s\n", "Ref", "Result", "State");
    printf("%s\n", "--------------------------------------------");

    for (int i = 0; i < n; i++) {
        int pg = pages[i];
        printf("Ref %2d: ", pg);

        if (page_hit(pg)) {
            printf("HIT\n");
        } else {
            faults++;
            printf("FAULT\n");
            if (empty_slots > 0) {
                /* Still have empty frames — fill directly */
                int slot = nf - empty_slots;
                frames[slot].page = pg;
                frames[slot].ref  = 1;
                empty_slots--;
            } else {
                clock_replace(pg);
            }
        }
        print_frames();
    }

    printf("\n=== Results ===\n");
    printf("Total page faults : %d\n", faults);
    printf("Hit ratio         : %.2f%%\n", (float)(n-faults)/n*100);
    return 0;
}

Enhanced Second-Chance (NRU) — C Code

NRU extends Clock with a dirty/modified bit (M). Prefer evicting clean pages over dirty ones, and unreferenced over referenced ones.

/*
 * NRU (Not Recently Used) / Enhanced Second-Chance Algorithm
 * -----------------------------------------------------------
 * Each frame has two bits: R (referenced) and M (modified/dirty).
 * Four classes (R,M): prefer lower class for eviction.
 *   Class 0 (0,0): best victim — not recent, not dirty
 *   Class 1 (0,1): not recent, dirty (must write to disk)
 *   Class 2 (1,0): recent, clean
 *   Class 3 (1,1): worst victim — recent AND dirty
 */

#include <stdio.h>
#include <string.h>
#include <stdlib.h>

#define MAX_F 10
#define MAX_P 100

typedef struct {
    int page;
    int R;     /* reference bit */
    int M;     /* modified (dirty) bit */
} NRUFrame;

NRUFrame fr[MAX_F];
int nf, hand_nru = 0;

/* Compute NRU class for frame j */
int nru_class(int j) {
    return (fr[j].R << 1) | fr[j].M;
    /* R=1,M=0 → class 2; R=0,M=1 → class 1; etc. */
}

int nru_find_victim() {
    /* Two passes over circular buffer looking for lowest class */
    for (int cls = 0; cls <= 3; cls++) {
        int ptr = hand_nru;
        do {
            if (nru_class(ptr) == cls) {
                hand_nru = (ptr + 1) % nf;
                return ptr;
            }
            ptr = (ptr + 1) % nf;
        } while (ptr != hand_nru);

        /* Didn't find class cls — clear R bits (simulate timer) */
        if (cls == 1) {    /* after scanning classes 0 and 1, clear R */
            for (int i = 0; i < nf; i++) fr[i].R = 0;
        }
    }
    return hand_nru;   /* fallback */
}

int main() {
    int pages[MAX_P], modify[MAX_P], n;

    printf("=== NRU (Enhanced Second-Chance) ===\n\n");
    printf("Frames: "); scanf("%d", &nf);
    printf("References: "); scanf("%d", &n);
    printf("Page refs (space-separated): ");
    for (int i = 0; i < n; i++) scanf("%d", &pages[i]);
    printf("Write? (0=read, 1=write) for each: ");
    for (int i = 0; i < n; i++) scanf("%d", &modify[i]);

    for (int i = 0; i < nf; i++) { fr[i].page=-1; fr[i].R=fr[i].M=0; }

    int faults=0, loaded=0;

    for (int i = 0; i < n; i++) {
        int pg = pages[i], hit = -1;
        for (int j=0; j<nf; j++) if (fr[j].page==pg) { hit=j; break; }

        if (hit != -1) {
            fr[hit].R = 1;
            if (modify[i]) fr[hit].M = 1;
            printf("Ref %2d (%s): HIT   class(%d,%d)=%d\n",
                   pg, modify[i]?"W":"R", fr[hit].R, fr[hit].M, nru_class(hit));
        } else {
            faults++;
            int slot;
            if (loaded < nf) slot = loaded++;
            else             slot = nru_find_victim();

            if (fr[slot].M)
                printf("  → dirty page %d written to disk\n", fr[slot].page);

            fr[slot].page = pg;
            fr[slot].R    = 1;
            fr[slot].M    = modify[i] ? 1 : 0;
            printf("Ref %2d (%s): FAULT → slot %d\n", pg, modify[i]?"W":"R", slot);
        }
    }
    printf("\nTotal page faults: %d | Hit ratio: %.1f%%\n",
           faults, (float)(n-faults)/n*100);
    return 0;
}

LFU & MFU Page Replacement

Keep a frequency counter per page. LFU evicts the least-frequently-used page; MFU evicts the most-frequently-used. Neither is used in mainstream OSes for page replacement, but they appear in database buffer pools and cache systems. Knowing their weaknesses helps understand why LRU-approximations win in practice.

/*
 * LFU (Least Frequently Used) and MFU (Most Frequently Used)
 * -----------------------------------------------------------
 * Both maintain a frequency counter for each frame.
 * LFU evicts: min frequency.
 * MFU evicts: max frequency.
 * Tie-breaking: use FIFO order (oldest loaded page wins).
 *
 * LFU weakness: "cache pollution" — startup pages accumulate
 *   high counts and resist eviction even when no longer needed.
 * MFU rationale: newly loaded pages haven't been used much;
 *   frequently used ones must be important — keep them.
 *
 * Compile: gcc lfu_mfu.c -o lfu_mfu
 */

#include <stdio.h>
#include <string.h>
#include <limits.h>

#define MAX_F 10
#define MAX_P 100

typedef struct {
    int page;     /* page number (-1 = empty)                */
    int freq;     /* access frequency counter                */
    int loaded;   /* timestamp when loaded (for tie-break)   */
} CountFrame;

CountFrame frames[MAX_F];
int nf;
int timer_cf = 0;  /* global clock for tie-breaking */

/* ── Find a page in frames; return index or -1 ── */
int find_page(int page) {
    for (int i = 0; i < nf; i++)
        if (frames[i].page == page) return i;
    return -1;
}

/* ── Find empty frame slot ── */
int find_empty() {
    for (int i = 0; i < nf; i++)
        if (frames[i].page == -1) return i;
    return -1;
}

/* ── LFU victim: min frequency; tie → oldest loaded ── */
int lfu_victim() {
    int vic = 0;
    for (int i = 1; i < nf; i++) {
        if (frames[i].freq < frames[vic].freq ||
           (frames[i].freq == frames[vic].freq &&
            frames[i].loaded < frames[vic].loaded))
            vic = i;
    }
    return vic;
}

/* ── MFU victim: max frequency; tie → oldest loaded ── */
int mfu_victim() {
    int vic = 0;
    for (int i = 1; i < nf; i++) {
        if (frames[i].freq > frames[vic].freq ||
           (frames[i].freq == frames[vic].freq &&
            frames[i].loaded < frames[vic].loaded))
            vic = i;
    }
    return vic;
}

/* ── Print current frame state ── */
void print_state() {
    printf("  [");
    for (int i = 0; i < nf; i++) {
        if (frames[i].page == -1) printf(" ---");
        else printf(" p%d(f=%d)", frames[i].page, frames[i].freq);
    }
    printf(" ]\n");
}

/* ── Run one algorithm ── */
int run(int pages[], int n, int use_lfu) {
    /* Initialise */
    for (int i = 0; i < nf; i++) {
        frames[i].page = -1;
        frames[i].freq = 0;
        frames[i].loaded = 0;
    }
    timer_cf = 0;
    int faults = 0;

    printf("\n=== %s (%d frames) ===\n", use_lfu ? "LFU" : "MFU", nf);
    printf("%-5s %-8s %s\n", "Page", "Result", "Frame State");

    for (int i = 0; i < n; i++) {
        int pg = pages[i];
        timer_cf++;
        int idx = find_page(pg);

        if (idx != -1) {
            /* HIT */
            frames[idx].freq++;
            printf("%-5d HIT    ", pg);
        } else {
            /* FAULT */
            faults++;
            int slot = find_empty();
            if (slot == -1) {
                slot = use_lfu ? lfu_victim() : mfu_victim();
                printf("      → evict page %d (freq=%d)\n",
                       frames[slot].page, frames[slot].freq);
            }
            frames[slot].page   = pg;
            frames[slot].freq   = 1;           /* fresh count */
            frames[slot].loaded = timer_cf;
            printf("%-5d FAULT  ", pg);
        }
        print_state();
    }
    printf("Page faults: %d | Hit ratio: %.1f%%\n",
           faults, (float)(n-faults)/n*100);
    return faults;
}

int main() {
    int pages[MAX_P], n;

    printf("=== LFU / MFU Page Replacement Simulator ===\n\n");
    printf("Number of frames: "); scanf("%d", &nf);
    printf("Number of refs  : "); scanf("%d", &n);
    printf("Page references : ");
    for (int i=0; i<n; i++) scanf("%d", &pages[i]);

    int f_lfu = run(pages, n, 1);   /* run LFU */
    int f_mfu = run(pages, n, 0);   /* run MFU */

    printf("\n=== Comparison ===\n");
    printf("LFU: %d faults\nMFU: %d faults\n", f_lfu, f_mfu);
    printf("Winner: %s\n", f_lfu < f_mfu ? "LFU" :
                              f_mfu < f_lfu ? "MFU" : "Tie");
    return 0;
}

 * ║   Page Replacement Algorithms — All-in-One Driver        ║
 * ║   Implements: FIFO | OPT | LRU | Clock | LFU            ║

#include <stdio.h>
#include <string.h>
#include <limits.h>
#include <stdbool.h>

#define MF  20    /* max frames  */
#define MP  200   /* max pages   */
#define INF 99999

int pages[MP], nf, np;

/* ─────────────────── Shared helpers ─────────────────── */
bool in_frames(int fr[], int pg) {
    for (int i=0;i<nf;i++) if(fr[i]==pg) return true;
    return false;
}
void print_bar(int faults, const char* name) {
    printf("  %-8s: %2d faults | %5.1f%% hit rate\n",
           name, faults, (float)(np-faults)/np*100);
}

/* ═══════════════════════════════════════
   1. FIFO — oldest page is evicted first
═══════════════════════════════════════ */
int fifo() {
    int fr[MF], ptr=0, faults=0;
    memset(fr,-1,sizeof(fr));
    for(int i=0;i<np;i++){
        if(!in_frames(fr,pages[i])){
            fr[ptr]=pages[i]; ptr=(ptr+1)%nf; faults++;
        }
    }
    return faults;
}

/* ═══════════════════════════════════════
   2. OPT — evict page used furthest ahead
═══════════════════════════════════════ */
int next_use(int pg, int from) {
    for(int k=from;k<np;k++) if(pages[k]==pg) return k;
    return INF;
}
int optimal() {
    int fr[MF], fc=0, faults=0;
    memset(fr,-1,sizeof(fr));
    for(int i=0;i<np;i++){
        if(in_frames(fr,pages[i])) continue;
        faults++;
        if(fc<nf){fr[fc++]=pages[i];continue;}
        int vic=0,mxd=-1;
        for(int j=0;j<nf;j++){
            int d=next_use(fr[j],i+1);
            if(d==INF){vic=j;break;}
            if(d>mxd){mxd=d;vic=j;}
        }
        fr[vic]=pages[i];
    }
    return faults;
}

/* ═══════════════════════════════════════
   3. LRU — evict least recently used page
═══════════════════════════════════════ */
int lru() {
    int fr[MF], lu[MF], fc=0, faults=0, t=0;
    memset(fr,-1,sizeof(fr));
    memset(lu,0,sizeof(lu));
    for(int i=0;i<np;i++){
        t++;
        int hi=-1;
        for(int j=0;j<nf;j++) if(fr[j]==pages[i]){hi=j;break;}
        if(hi!=-1){lu[hi]=t;continue;}
        faults++;
        int slot = fc<nf ? fc++ : 0;
        if(fc==nf){
            int mn=lu[0];
            for(int j=1;j<nf;j++) if(lu[j]<mn){mn=lu[j];slot=j;}
        }
        fr[slot]=pages[i]; lu[slot]=t;
    }
    return faults;
}

/* ═══════════════════════════════════════
   4. CLOCK — second-chance algorithm
═══════════════════════════════════════ */
int clock_algo() {
    int  fr[MF], ref[MF], hand=0, faults=0, fc=0;
    memset(fr,-1,sizeof(fr));
    memset(ref,0,sizeof(ref));
    for(int i=0;i<np;i++){
        int hit=-1;
        for(int j=0;j<nf;j++) if(fr[j]==pages[i]){hit=j;break;}
        if(hit!=-1){ref[hit]=1;continue;}
        faults++;
        if(fc<nf){
            fr[fc]=pages[i]; ref[fc]=1; fc++;
        }else{
            while(ref[hand]){ref[hand]=0;hand=(hand+1)%nf;}
            fr[hand]=pages[i]; ref[hand]=1; hand=(hand+1)%nf;
        }
    }
    return faults;
}

/* ═══════════════════════════════════════
   5. LFU — evict least frequently used
═══════════════════════════════════════ */
int lfu() {
    int fr[MF], freq[MF], load[MF], fc=0, faults=0, t=0;
    memset(fr,-1,sizeof(fr));
    memset(freq,0,sizeof(freq));
    memset(load,0,sizeof(load));
    for(int i=0;i<np;i++){
        t++;
        int hi=-1;
        for(int j=0;j<nf;j++) if(fr[j]==pages[i]){hi=j;break;}
        if(hi!=-1){freq[hi]++;continue;}
        faults++;
        int slot;
        if(fc<nf) slot=fc++;
        else{
            slot=0;
            for(int j=1;j<nf;j++)
                if(freq[j]<freq[slot]||
                  (freq[j]==freq[slot]&&load[j]<load[slot]))
                    slot=j;
        }
        fr[slot]=pages[i]; freq[slot]=1; load[slot]=t;
    }
    return faults;
}

💡 Why Clock beats LRU in this example

Clock (11 faults) < LRU (12 faults) here — a quirk of this specific reference string. In general, LRU ≤ Clock ≤ FIFO for most workloads. Clock's approximation sometimes "gets lucky" with its sweep order. The key point: Clock is O(1) per reference while true LRU needs O(n) or a complex data structure.
