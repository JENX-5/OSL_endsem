🧠
The Big Picture
why memory management exists at all
☕ Analogy First

Imagine a restaurant (your RAM) with limited seats (memory). Customers (processes) arrive and need a table (memory space). The waiter (OS memory manager) has to decide: which table do you give them? What if all tables are taken? What if a big party needs 6 seats but the restaurant only has scattered tables of 2? That's memory management in a nutshell — fitting processes into limited physical memory as efficiently as possible.

When you run multiple programs simultaneously, your OS needs to keep each program's code and data in RAM. But RAM is finite. The memory manager's job is to:

  • Keep track of which parts of RAM are in use and which are free
  • Allocate memory to processes when they request it
  • Reclaim memory when processes finish
  • Protect one process from accidentally accessing another's memory
  • Give processes the illusion of having more memory than physically exists (virtual memory)
Address Binding

Translating symbolic addresses in your code to actual physical memory addresses. Happens at compile time, load time, or runtime.

Logical vs Physical Address

Logical address = what the CPU generates. Physical address = actual RAM location. The MMU (Memory Management Unit) does the translation.

MMU

Hardware chip that translates logical addresses to physical ones. With relocation register: Physical = Logical + Base.

💡 Key Insight

In modern OSes, processes never see physical addresses directly. They always work with logical (virtual) addresses. This abstraction is what makes memory protection and virtual memory possible.

🏗️
Memory Partitioning
Fixed-size vs Variable-size Partitions

The earliest and simplest approach to memory management. Divide RAM into chunks, give each chunk to a process.

Fixed-Size Partitioning

☕ Analogy

Imagine a parking lot with fixed-size parking spaces. A small car (small process) gets the same space as a truck (large process). Wasteful? Yes. Simple? Absolutely. That's fixed-size partitioning.

  • Memory is divided into N partitions of equal (or different fixed) sizes at boot time
  • Each partition can hold exactly ONE process at a time
  • The OS keeps a partition table tracking which are free/occupied
  • Maximum degree of multiprogramming = N (number of partitions)
⚠️ Problem: Internal Fragmentation

A 50KB process in a 100KB partition wastes 50KB. That wasted space inside a partition is called internal fragmentation. You paid for a double room but sleep alone.

Variable-Size Partitioning (Dynamic Partitioning)

Instead of pre-defined sizes, the OS creates partitions of exactly the size needed by each process. Much more flexible — but introduces a new problem.

  • OS maintains a list of free memory blocks called holes
  • When a process arrives, the OS allocates a hole big enough for it
  • When a process leaves, its memory is returned to the free list (becomes a hole)
  • Adjacent free holes can be merged (coalescing)
  • Problem: Over time, free memory gets scattered in small, non-contiguous chunks → External Fragmentation
💡 Coalescing

When a process releases memory, the OS checks if adjacent memory blocks are also free. If so, it merges them into one bigger hole. This reduces fragmentation — but doesn't eliminate it completely.

🔍
Dynamic Storage Allocation
The "which hole do I pick?" problem

When multiple free holes exist and a new process needs memory, the OS must choose which hole to use. This is the Dynamic Storage Allocation Problem. Three classic solutions:

Strategy How It Works Speed Memory Use Leftover
First Fit Pick the FIRST hole big enough Fastest Medium Larger holes
Best Fit Pick the SMALLEST hole big enough Slowest Best Smallest
Worst Fit Pick the LARGEST hole Slow Worst Largest (usable)
🧠 Counterintuitive Insight: Why Worst Fit?

Worst Fit might seem… terrible. But the logic is: if you always take the biggest hole, you leave a big leftover chunk that future large processes can still use. Best Fit's tiny leftover holes are often too small to be useful — creating "useless slivers" scattered everywhere.

⚙️
First Fit / Best Fit / Worst Fit
Algorithms, pseudocode, and visual traces

First Fit — Algorithm

1
Maintain a list of holes

The OS keeps track of all free memory blocks with their starting address and size.

2
For each incoming process

Scan the hole list from the beginning (or from where the last search ended — "next fit" variant).

3
Pick the first hole ≥ process size

Stop immediately upon finding one. Don't look further.

4
Allocate & split

Give the process exactly what it needs. The remainder (hole size − process size) becomes a new, smaller hole.

5
If no hole is big enough

Process waits (or compaction is triggered). Report "allocation failed."

Best Fit — Algorithm

1
Scan the entire hole list

Cannot stop early — must check every hole to find the minimum-sized sufficient one.

2
Track the best candidate

Keep a pointer to the hole where: hole_size ≥ process_size AND hole_size is minimum.

3
Allocate from the best candidate

Split as in First Fit. The leftover fragment will be tiny (best case: zero waste).

4
Optimization: sorted list

If holes are kept sorted by size, Best Fit degrades to O(log n) search instead of O(n).

Worst Fit — Algorithm

Identical to Best Fit except in step 2: track the maximum-sized hole instead of minimum.

📊 Performance Comparison (Simulations show)

Both First Fit and Best Fit are better than Worst Fit in terms of overall memory utilization. First Fit tends to be faster and good enough in practice. Best Fit saves slightly more memory but produces more tiny unusable fragments.

💥
Fragmentation
The memory manager's nightmare
External Fragmentation

Total free memory is enough for a process, but it's scattered in small non-contiguous chunks. The process can't use it because it needs contiguous space.

Internal Fragmentation

Allocated memory is slightly larger than requested. The extra (unused) space inside the partition is wasted. Common in fixed-size partitioning and paging.

Compaction — The Solution to External Fragmentation

☕ Analogy

It's like defragmenting a hard drive, or reorganizing your wardrobe by shoving all clothes to one side so there's one big empty space instead of several small gaps. You physically move things to make room.

  • Shuffle memory contents to consolidate all free space into one large contiguous block
  • Very expensive — requires copying entire process address spaces
  • Only possible if relocation is done at runtime (dynamic binding)
  • During compaction, all processes must be paused → significant performance hit
  • Modern systems prefer paging instead, which eliminates external fragmentation entirely
📐 The 50% Rule (Knuth)

Statistical analysis shows that with First Fit allocation, roughly ½ of N allocated blocks are lost to fragmentation — i.e., for every 3 usable partitions, 1 is wasted to fragmentation. This is why the OS needs better schemes like paging.

📄
Paging
The revolutionary fix for external fragmentation
☕ Analogy

Imagine a book (process) that needs space on shelves (RAM). Instead of needing one long contiguous shelf space, you tear the book into equal-sized chapters (pages) and put each chapter on any available shelf slot (frame). A table of contents (page table) tells you which slot holds which chapter. You can read the book in order even though the chapters are scattered — because you consult the TOC each time.

Core Idea

  • Divide physical memory into fixed-size blocks called frames
  • Divide logical memory into blocks of the same size called pages
  • When a process runs, load its pages into any available frames (they don't have to be contiguous!)
  • The page table records which frame holds each page
  • The CPU generates logical addresses; the MMU translates them using the page table

Address Translation

Every logical address has two parts:

Field Meaning Bits
Page Number (p) Index into the page table — tells us which page high-order bits
Page Offset (d) Byte position within the page/frame low-order bits
Physical Address = (frame_number × page_size) + offset If logical address = X, page_size = 2^n: page = X / page_size = X >> n offset = X mod page_size = X & (page_size - 1)

How Many Bits for What?

  • If logical address space = 2m bytes and page size = 2n bytes:
  • Page number uses m − n bits
  • Page offset uses n bits
  • Number of pages = 2(m−n)
📐 Lab Exercise: Page size = 32 bytes, 8 pages, 256 bytes total
Setup Total memory = 256 bytes = 28 → 8-bit address. Page size = 32 = 25 → 5-bit offset. Page number = 8−5 = 3 bits. So address format: [p p p | d d d d d]
204 bytes 204 in binary = 11001100. Split: page bits = 110 = 6, offset bits = 01100 = 12. Page 6, Offset 12
56 bytes 56 in binary = 00111000. Split: page bits = 001 = 1, offset bits = 11000 = 24. Page 1, Offset 24
Bits needed We need 8 bits to represent all 256 addresses (0–255).

Page Table — Where Does It Live?

  • In registers: Fast but only works for tiny page tables (few pages)
  • In main memory: Works for large tables, but every memory access now requires 2 RAM accesses (one for page table, one for data) — slow!
  • TLB (Translation Lookaside Buffer): A small, super-fast hardware cache for page table entries. Typical hit rate: 99%+. With TLB hit, translation is nearly free.
💡 TLB Miss Handling

On a TLB miss, the hardware (or OS) walks the page table in RAM to find the mapping, loads it into the TLB, then retransmits the memory access. This is called a "page table walk." Modern CPUs have hardware page table walkers (PTWs) that do this automatically.

Protection in Paging

  • Each page table entry includes protection bits: read/write/execute permissions
  • Accessing a page you don't have permission for → hardware trap → segmentation fault
  • Valid/Invalid bit: If a page isn't in memory (swapped out), its entry has the invalid bit set. Accessing it triggers a page fault — the OS loads the page from disk.

Shared Pages

  • Multiple processes can share the same physical frames by having their page tables point to the same frame numbers
  • Perfect for shared libraries (libc, etc.) — code pages are read-only and can be shared across all processes using that library
  • This is the foundation of memory-mapped files and copy-on-write

Problems With Simple Paging

  • Page table size: For 32-bit address space with 4KB pages → 220 = 1 million entries! At 4 bytes each = 4MB just for page table, per process
  • Solution: Hierarchical page tables (multi-level), hashed page tables, inverted page tables
  • Internal fragmentation still exists: last page of a process may not be full
🔬 Advanced: Multilevel Paging

In x86-64, there are 4 levels of page tables (PML4 → PDPT → PD → PT). Each level maps a portion of the virtual address. This way, most of the page table can be swapped out — only the parts corresponding to active address ranges need to be in memory.

🗂️
Segmentation
memory that matches how programmers think
☕ Analogy

Paging splits memory blindly by equal chunk size. Segmentation is smarter — it splits memory by meaning. Your program has a code section, a stack, a heap, a data section. Segmentation gives each one its own logical chunk in memory. It's like organizing your house by rooms (kitchen, bedroom, office) rather than arbitrary 10m² grids.

Key Concepts

  • A program is divided into logical segments: code, data, stack, heap, etc.
  • Each segment has a name and a length (offset range 0 to length−1)
  • Logical address = ⟨segment number, offset⟩
  • The Segment Table maps each segment to its physical location
  • Each segment table entry has a base (starting physical address) and limit (segment size)

Address Translation in Segmentation

1
Extract segment number s and offset d from logical address

CPU generates the ⟨s, d⟩ pair.

2
Look up Segment Table[s]

Get base[s] and limit[s].

3
Bounds check: d < limit[s]?

If not: trap! → segmentation fault. Your program tried to access beyond its segment boundary.

4
Physical Address = base[s] + d

Simple addition gives the physical address.

Segmentation vs. Paging

Feature Paging Segmentation
Division Fixed-size pages (transparent to programmer) Variable-size segments (matches program structure)
External Fragmentation None Yes
Internal Fragmentation Small (last page) None
Sharing Harder (must align on page boundaries) Natural (share whole segments)
Protection Per-page Per-segment (natural boundaries)
Hardware complexity Medium Higher
💡 Real World: Segmentation + Paging Combined

x86 processors actually have both! Each segment maps to a range of linear addresses, which are then translated by paging. In practice, modern OSes (Linux, Windows) use "flat segmentation" (all segments overlap, covering the full address space) and rely only on paging for the real work. Segmentation is effectively vestigial on x86-64 in user mode.

First Fit, Best Fit & Worst Fit in C

C — memory_allocation.c 
/*
 * Memory Allocation Strategies: First Fit, Best Fit, Worst Fit
 * Lab 9 - OS Memory Management
 * Compile: gcc memory_allocation.c -o memory_allocation
 */

#include <stdio.h>
#include <string.h>

#define MAX_HOLES  10
#define MAX_PROCS  10

/* ─── Data Structures ─── */

typedef struct {
    int id;           /* Hole identifier (1-based) */
    int size;         /* Original hole size */
    int remaining;    /* Remaining size after allocations */
} Hole;

typedef struct {
    int id;           /* Process identifier (1-based) */
    int size;         /* Memory required */
    int hole_id;      /* Which hole was assigned (-1 = none) */
} Process;

/* ─── Helper: Reset allocations ─── */
void reset(Hole holes[], int nh, Process procs[], int np) {
    for (int i = 0; i < nh; i++)
        holes[i].remaining = holes[i].size;   /* restore original sizes */
    for (int i = 0; i < np; i++)
        procs[i].hole_id = -1;              /* mark all as unallocated */
}

/* ─── Print Results ─── */
void print_results(const char* strategy, Hole holes[], int nh, Process procs[], int np) {
    printf("\n╔══════════════════════════════════════╗\n");
    printf("  %-38s\n", strategy);
    printf("╚══════════════════════════════════════╝\n");

    printf("\n  PROCESS ALLOCATION:\n");
    printf("  %-12s %-12s %s\n", "Process", "Required", "Hole Assigned");
    printf("  %-12s %-12s %s\n", "-------", "--------", "-------------");
    for (int i = 0; i < np; i++) {
        if (procs[i].hole_id == -1)
            printf("  P%-11d %-12d Not allocated\n", procs[i].id, procs[i].size);
        else
            printf("  P%-11d %-12d Hole %d\n", procs[i].id, procs[i].size, procs[i].hole_id);
    }

    printf("\n  HOLE STATUS:\n");
    printf("  %-8s %-12s %-12s %s\n", "Hole", "Original", "Remaining", "Used");
    printf("  %-8s %-12s %-12s %s\n", "----", "--------", "---------", "----");
    for (int i = 0; i < nh; i++) {
        printf("  %-8d %-12d %-12d %d KB\n",
            holes[i].id, holes[i].size, holes[i].remaining,
            holes[i].size - holes[i].remaining);
    }
}

/* ═══════════════════════════════════════
   FIRST FIT ALGORITHM
   - Scan holes in order, pick the FIRST
     one big enough.
═══════════════════════════════════════ */
void first_fit(Hole holes[], int nh, Process procs[], int np) {
    for (int i = 0; i < np; i++) {          /* iterate each process */
        for (int j = 0; j < nh; j++) {       /* iterate holes in order */
            if (holes[j].remaining >= procs[i].size) {
                procs[i].hole_id   = holes[j].id;
                holes[j].remaining -= procs[i].size;  /* reduce remaining space */
                break;                               /* STOP - first fit found */
            }
        }
    }
    print_results("FIRST FIT", holes, nh, procs, np);
}

/* ═══════════════════════════════════════
   BEST FIT ALGORITHM
   - Scan ALL holes, pick the SMALLEST
     one that's big enough.
═══════════════════════════════════════ */
void best_fit(Hole holes[], int nh, Process procs[], int np) {
    for (int i = 0; i < np; i++) {
        int best_j = -1;
        int best_diff = 999999;             /* track smallest sufficient hole */

        for (int j = 0; j < nh; j++) {
            int diff = holes[j].remaining - procs[i].size;
            if (diff >= 0 && diff < best_diff) {
                best_diff = diff;
                best_j = j;               /* update best candidate */
            }
        }
        if (best_j != -1) {
            procs[i].hole_id = holes[best_j].id;
            holes[best_j].remaining -= procs[i].size;
        }
    }
    print_results("BEST FIT", holes, nh, procs, np);
}

/* ═══════════════════════════════════════
   WORST FIT ALGORITHM
   - Scan ALL holes, pick the LARGEST one.
═══════════════════════════════════════ */
void worst_fit(Hole holes[], int nh, Process procs[], int np) {
    for (int i = 0; i < np; i++) {
        int worst_j = -1;
        int worst_size = -1;              /* track largest hole */

        for (int j = 0; j < nh; j++) {
            if (holes[j].remaining >= procs[i].size
                && holes[j].remaining > worst_size) {
                worst_size = holes[j].remaining;
                worst_j = j;
            }
        }
        if (worst_j != -1) {
            procs[i].hole_id = holes[worst_j].id;
            holes[worst_j].remaining -= procs[i].size;
        }
    }
    print_results("WORST FIT", holes, nh, procs, np);
}

/* ═══════════════════════════════════════
   MAIN — drives the simulation
═══════════════════════════════════════ */
int main() {
    Hole holes[MAX_HOLES];
    Process procs[MAX_PROCS];
    int nh, np;

    printf("=== Memory Allocation Strategy Simulator ===\n\n");

    printf("Enter number of holes: ");
    scanf("%d", &nh);
    for (int i = 0; i < nh; i++) {
        holes[i].id = i + 1;
        printf("  Size of Hole %d (KB): ", i + 1);
        scanf("%d", &holes[i].size);
        holes[i].remaining = holes[i].size;
    }

    printf("\nEnter number of processes: ");
    scanf("%d", &np);
    for (int i = 0; i < np; i++) {
        procs[i].id = i + 1;
        printf("  Memory required by P%d (KB): ", i + 1);
        scanf("%d", &procs[i].size);
        procs[i].hole_id = -1;
    }

    /* Run all three strategies independently */
    reset(holes, nh, procs, np);
    first_fit(holes, nh, procs, np);

    reset(holes, nh, procs, np);
    best_fit(holes, nh, procs, np);

    reset(holes, nh, procs, np);
    worst_fit(holes, nh, procs, np);

    return 0;
}

C — paging_simulation.c 
/*
 * Paging - Logical to Physical Address Translation
 * Lab 9 - OS Memory Management
 */

#include <stdio.h>
#include <math.h>

#define MAX_PAGES  256   /* max number of pages */

int page_table[MAX_PAGES];  /* page_table[page_num] = frame_num */

/* ─── Translate a logical address to physical ─── */
void translate(int logical, int page_size) {
    int page_num  = logical / page_size;       /* integer division → page number */
    int offset    = logical % page_size;       /* remainder → offset within page */
    int frame_num = page_table[page_num];      /* look up physical frame */
    int physical  = (frame_num * page_size) + offset;  /* compute physical addr */

    printf("\n  Logical Address   : %d\n", logical);
    printf("  Page Number (p)  : %d  (binary: ", page_num);
    /* print binary representation */
    for (int b = 7; b >= 0; b--)
        printf("%d", (page_num >> b) & 1);
    printf(")\n");
    printf("  Page Offset (d)  : %d\n", offset);
    printf("  Frame Number     : %d\n", frame_num);
    printf("  Physical Address : %d\n", physical);
}

int main() {
    int page_size, num_pages, logical_addr;

    printf("=== Paging Address Translator ===\n\n");
    printf("Enter page size (bytes): ");
    scanf("%d", &page_size);

    printf("Enter number of pages: ");
    scanf("%d", &num_pages);

    /* Build the page table — enter frame numbers */
    printf("\nEnter page table (page → frame mapping):\n");
    for (int i = 0; i < num_pages; i++) {
        printf("  Page %d → Frame: ", i);
        scanf("%d", &page_table[i]);
    }

    /* Translate addresses */
    printf("\nEnter logical address to translate (-1 to quit): ");
    while (scanf("%d", &logical_addr) && logical_addr != -1) {
        int total_logical = page_size * num_pages;
        if (logical_addr < 0 || logical_addr >= total_logical) {
            printf("  ERROR: Address %d out of logical space [0, %d)\n",
                   logical_addr, total_logical);
        } else {
            translate(logical_addr, page_size);
        }
        printf("\nEnter logical address (-1 to quit): ");
    }
    return 0;
}

/*
 * Segmentation - Logical to Physical Address Translation
 * Lab 9 Additional Exercise
 */

#include <stdio.h>

#define MAX_SEGS 10

/* Each segment table entry has a base and a limit */
typedef struct {
    int base;     /* starting physical address */
    int limit;    /* size/length of segment */
} SegmentEntry;

SegmentEntry seg_table[MAX_SEGS];

/* ─── Translate ⟨segment, offset⟩ → physical ─── */
void translate(int seg, int offset, int num_segs) {
    /* Step 1: check segment number is valid */
    if (seg < 0 || seg >= num_segs) {
        printf("  TRAP: Invalid segment number %d\n", seg);
        return;
    }

    /* Step 2: bounds check — offset must be within segment */
    if (offset < 0 || offset >= seg_table[seg].limit) {
        printf("  TRAP: Offset %d out of bounds for Segment %d"
               " (limit=%d)\n",
               offset, seg, seg_table[seg].limit);
        printf("  → SEGMENTATION FAULT!\n");
        return;
    }

    /* Step 3: compute physical address */
    int physical = seg_table[seg].base + offset;

    printf("  Logical ⟨%d, %d⟩ → Physical: %d\n",
           seg, offset, physical);
    printf("    (base[%d]=%d + offset=%d)\n",
           seg, seg_table[seg].base, offset);
}

int main() {
    int num_segs;

    printf("=== Segmentation Address Translator ===\n\n");
    printf("Enter number of segments: ");
    scanf("%d", &num_segs);

    /* Build segment table */
    printf("\nEnter segment table:\n");
    for (int i = 0; i < num_segs; i++) {
        printf("  Segment %d — Base: ", i);
        scanf("%d", &seg_table[i].base);
        printf("  Segment %d — Limit: ", i);
        scanf("%d", &seg_table[i].limit);
    }

    /* Translate addresses */
    int seg, offset;
    printf("\nEnter logical addresses ⟨segment offset⟩ (-1 to quit):\n");
    while (1) {
        printf("  Segment: ");
        scanf("%d", &seg);
        if (seg == -1) break;
        printf("  Offset : ");
        scanf("%d", &offset);
        translate(seg, offset, num_segs);
        printf("\n");
    }
    return 0;
}

Next-Fit Allocation in C

A variant of First Fit that remembers where the last search ended, distributing allocations more evenly across memory.

C — next_fit.c 
/*
 * Next-Fit Memory Allocation
 * ─────────────────────────────────────────────────
 * Like First Fit, but starts scanning from where the
 * LAST allocation ended rather than always from index 0.
 * Spreads allocations more evenly; avoids clustering
 * small holes at the beginning of memory.
 *
 * Compile: gcc next_fit.c -o next_fit
 */

#include <stdio.h>
#include <string.h>

#define MAX_H 20
#define MAX_P 20

typedef struct { int size, rem; } Hole;
typedef struct { int size, hole; } Proc;

int nh, np;
Hole holes[MAX_H];
Proc procs[MAX_P];

void next_fit() {
    int last = 0;   /* remember where we left off */

    printf("\n=== NEXT FIT ===\n");
    for (int i = 0; i < np; i++) {
        int found = 0;

        /* Scan from 'last' wrapping around circularly */
        for (int cnt = 0; cnt < nh; cnt++) {
            int j = (last + cnt) % nh;   /* circular indexing */
            if (holes[j].rem >= procs[i].size) {
                procs[i].hole  = j + 1;
                holes[j].rem  -= procs[i].size;
                last           = j;            /* update scan start */
                found          = 1;
                printf("P%d (%dK) → Hole %d (rem %dK)\n",
                       i+1, procs[i].size, j+1, holes[j].rem);
                break;
            }
        }
        if (!found) {
            procs[i].hole = -1;
            printf("P%d (%dK) → Not allocated\n", i+1, procs[i].size);
        }
    }
}

int main() {
    printf("Next-Fit Memory Allocation\n\n");
    printf("Number of holes   : "); scanf("%d", &nh);
    for (int i = 0; i < nh; i++) {
        printf("  Hole %d size (K): ", i+1); scanf("%d", &holes[i].size);
        holes[i].rem = holes[i].size;
    }
    printf("Number of processes: "); scanf("%d", &np);
    for (int i = 0; i < np; i++) {
        printf("  P%d size (K): ", i+1); scanf("%d", &procs[i].size);
        procs[i].hole = -1;
    }
    next_fit();
    return 0;
}

/* KEY DIFFERENCE from First Fit:
 * First Fit always starts from hole[0].
 * Next Fit starts from hole[last_used].
 * If holes = [100K,500K,200K,300K,600K] and last=500K:
 *   Next request scans 200K→300K→600K first, NOT 100K.
 * Benefit: avoids wearing out the first few holes. */

Address Binding Stages — C Demo

Demonstrates compile-time, load-time, and run-time address binding with a relocation register simulation.

C — address_binding.c 
/*
 * Address Binding Simulation
 * ─────────────────────────────────────────────────
 * Shows how logical addresses are translated to
 * physical addresses using a relocation register
 * (base register) — the simplest form of MMU.
 *
 * Also demonstrates limit register for protection:
 * if logical addr >= limit → trap (segfault).
 *
 * Compile: gcc address_binding.c -o addr_bind
 */

#include <stdio.h>
#include <stdbool.h>

/* ─── Simulated MMU registers ─── */
typedef struct {
    int base;     /* relocation register: starting physical addr */
    int limit;    /* limit register: max logical address allowed  */
} MMU;

/* Translate logical → physical; check limit register */
bool translate(MMU mmu, int logical, int* physical) {
    if (logical < 0 || logical >= mmu.limit) {
        printf("  TRAP: logical %d >= limit %d → Segfault!\n",
               logical, mmu.limit);
        return false;
    }
    *physical = mmu.base + logical;
    return true;
}

void demo_process(const char* name, MMU mmu, int addrs[], int n) {
    printf("\n── %s (base=%d, limit=%d) ──\n",
           name, mmu.base, mmu.limit);
    printf("  %-12s  %-14s  %s\n",
           "Logical", "Physical", "Status");
    for (int i = 0; i < n; i++) {
        int phys;
        printf("  %-12d  ", addrs[i]);
        if (translate(mmu, addrs[i], &phys))
            printf("%-14d  ✓ OK\n", phys);
        /* trap message printed inside translate() on failure */
    }
}

int main() {
    printf("=== MMU Address Binding Simulation ===\n");
    printf("Physical Address = Base + Logical\n");
    printf("Protection: Logical must be < Limit\n");

    /* Process A: loaded at physical address 14000, size 10000 */
    MMU mmuA = { .base = 14000, .limit = 10000 };
    int addrsA[] = { 0, 346, 1752, 9999, 10000 };  /* last is illegal */
    demo_process("Process A", mmuA, addrsA, 5);

    /* Process B: loaded at physical address 30000, size 5000 */
    MMU mmuB = { .base = 30000, .limit = 5000  };
    int addrsB[] = { 0, 2000, 4999, 5000 };
    demo_process("Process B", mmuB, addrsB, 4);

    /* Interactive mode */
    printf("\n=== Interactive Mode ===\n");
    int base, limit;
    printf("Enter base register  : "); scanf("%d", &base);
    printf("Enter limit register : "); scanf("%d", &limit);
    MMU mmu = { base, limit };

    int la;
    printf("Enter logical address (-1 to quit): ");
    while (scanf("%d", &la) && la != -1) {
        int pa;
        if (translate(mmu, la, &pa))
            printf("  Physical address = %d\n", pa);
        printf("Enter logical address (-1 to quit): ");
    }
    return 0;
}

/* SAMPLE OUTPUT:
 * Process A (base=14000, limit=10000)
 *   Logical       Physical        Status
 *   0             14000           ✓ OK
 *   346           14346           ✓ OK
 *   1752          15752           ✓ OK
 *   9999          23999           ✓ OK
 *   TRAP: logical 10000 >= limit 10000 → Segfault!
 */

Common Mistakes
what students get wrong in exams
😬
Confusing Internal vs External Fragmentation Internal = wasted space INSIDE an allocated partition. External = total free space is enough but it's scattered. Fixed-size → internal. Variable-size → external.
😬
Stopping Best/Worst Fit early Best Fit and Worst Fit MUST scan the entire hole list. You cannot stop early. Only First Fit stops at the first match.
😬
Mixing up page number and frame number Page number comes from the logical address. Frame number comes from the page table lookup. They're different things. Physical address uses the frame number.
😬
Forgetting the bounds check in segmentation Always check offset < limit BEFORE computing physical address. If it fails, it's a trap/fault — don't compute a physical address at all.
😬
Thinking paging eliminates ALL fragmentation Paging eliminates EXTERNAL fragmentation. It still has INTERNAL fragmentation (the last page of a process may be partially empty).
😬
Page size must be power of 2 Page sizes are always powers of 2 (512B, 4KB, 2MB, 1GB). This makes address splitting trivial using bit shifts. Exam questions sometimes test this.

Viva & Exam Q&A
click any question to reveal the answer

Short Answer (Click to Expand)

▶What is the difference between First Fit and Best Fit?
First Fit allocates from the first hole big enough (scanning from the start, stops immediately). Best Fit scans the entire hole list and picks the smallest hole that's still big enough. First Fit is faster; Best Fit wastes less space per allocation but may create tiny unusable fragments.
▶What is external fragmentation? Give an example.
External fragmentation occurs when total free memory is enough for a process, but it's scattered in non-contiguous chunks. Example: You need 300KB. Holes available: 100KB, 120KB, 80KB (total = 300KB). But since the process needs a contiguous block, it can't be allocated — even though enough total free memory exists.
▶How does paging solve external fragmentation?
Paging divides a process into fixed-size pages and places them in any available frames — they don't need to be contiguous. So even if free frames are scattered throughout RAM, any process can be loaded. The page table keeps track of where each page lives. External fragmentation: completely eliminated.
▶What is a TLB and why is it needed?
TLB (Translation Lookaside Buffer) is a small, fast hardware cache for page table entries. Without it, every memory access requires two RAM accesses: one to read the page table, one for actual data. With TLB, if the mapping is cached (TLB hit, ~99% of time), translation costs nearly zero. It's essentially a cache for virtual-to-physical address mappings.
▶What are the two fields in a segmentation logical address?
A segmentation logical address has two components: ⟨segment number s, offset d⟩. The segment number indexes into the segment table to get the base and limit. The offset is the byte position within that segment. Physical address = base[s] + d, provided d < limit[s].
▶Why is compaction expensive?
Compaction requires physically moving process data in memory to consolidate free holes. This involves: copying potentially large amounts of data, updating all pointers and addresses (only possible with runtime relocation), and halting all processes during the operation. It has O(n) time and I/O cost. Modern systems avoid it by using paging instead.
▶🎯 Trick Question: Does paging have any fragmentation at all?
Yes! Paging eliminates external fragmentation but still has internal fragmentation. The last page of a process is rarely exactly full — the leftover space within that last page is wasted. On average, half a page per process is wasted (internal fragmentation = page_size/2 per process on average).
▶🎯 Trick Question: Which is better — First Fit or Best Fit?
Neither is universally better. Simulations show they perform similarly overall. First Fit is faster (no full scan needed). Best Fit wastes less per allocation but produces more tiny fragments that are useless. Best Fit can outperform First Fit for specific workloads (as shown in the lab exercise where Best Fit fits P4 but First Fit can't). In practice, First Fit is commonly preferred due to speed.
▶Long Answer: Compare paging and segmentation in detail.
Paging: Divides logical and physical memory into equal fixed-size units (pages/frames). Transparent to the programmer. Eliminates external fragmentation. Still has internal fragmentation (last page). Page table can be large. Hardware support via TLB makes it fast. Used in all modern OSes.

Segmentation: Divides memory according to the logical structure of the program (code, data, stack). Variable-size segments match programmer's view. No internal fragmentation. Has external fragmentation. Logical structure enables natural sharing and protection per segment. Segment table has base+limit per entry.

Combined (Intel x86): Logical address → segmentation → linear address → paging → physical address. Modern Linux/Windows use flat segmentation (all segments = full address space) effectively making segmentation a no-op, relying solely on paging.

Quick Revision Summary
everything you need for last-minute prep

Fixed Partitioning

Memory divided into fixed chunks at boot. One process per partition. Causes internal fragmentation. Simple but wasteful.

Variable Partitioning

Partitions created on demand = exact process size. Causes external fragmentation. Needs coalescing of free holes.

First Fit

Scan from start, pick first hole ≥ process size. Fastest (O(n) worst, often less). Generally good in practice.

Best Fit

Scan all holes, pick smallest sufficient one. Minimizes waste per allocation. Creates tiny unusable fragments. O(n) always.

Worst Fit

Pick largest hole. Leaves big remainders. Performs worst overall. Only useful when large leftover chunks are specifically needed.

Paging

Fixed-size pages = frames. Page table maps logical→physical. Eliminates external fragmentation. Has internal fragmentation. TLB speeds up translation.

Address in Paging

page = addr ÷ page_size | offset = addr mod page_size | physical = frame × page_size + offset

Segmentation

Variable-size segments match program structure. Logical addr = ⟨s, d⟩. Check d < limit[s] → physical = base[s] + d. External fragmentation still exists.

Compaction

Shuffle memory to consolidate free space. Fixes external fragmentation but very expensive. Requires runtime relocation support.

Fragmentation Quick Test

Wasted space INSIDE allocation → Internal. Enough total free space but non-contiguous → External. Paging → eliminates external, has internal. Segmentation → has external, no internal.
